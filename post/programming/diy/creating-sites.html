DIY.automate(): Making a site that makes sites

June 30, 2021

If you're working with pure HTML and CSS for your site (#VanillaHTML #Boo3rdPartyPlugins amiright) and it has an actual header and footer that takes up at least 25 lines of code PLUS meta tags on the <head> that takes an additional 5+ lines, then it's probably not ideal for you to start from scratch every time.

But using a template isn't also ideal for making pages with an insane amount of code that's being reused. Why? Because creating a site isn't just about creating the content of that site, but the location of it is one of the important things too. So you wouldn't want to waste your time finding src's and href's in your site and adding an endless amount of "../" to them, right?

So what tool do you use? Well, a site that creates a site for you!

How does a site maker work?

Well, you input stuff on inputtable stuff and magic happens and you get the output! Easy, right? Well, I guess the magic is JavaScript. But did you know that by hooking up a bunch of <textarea>'s you'll be able to easily create your own site maker? This page will guide you to it! (I wasn't originally planning for this but just showing my File maker won't help anyone so here's a guide!)

Requirements for building a site maker that builds full-fledged site

For a site to be technically called "full-fledged" or more commonly known as "why did you do this it's so overkill for a locally-developed website pls stop" then you need this stuff present in your site (at least in my interpretation): having an insane goal in your mind, having sufficient meta tags (including the ones that social media APIs look for), and having different parts of your site like headers and footers. So, you'll need these things:

  • Lots of programming and crying
  • A title for your og:name and title
  • The site description for both meta description and og:description
  • The location of the page relative to the root of your site
  • If you want, you can even add a "date posted" part
  • And finally, the actual content of the site

Oh yeah, if you don't know what the og: stuff here means, I recommend you to check out The Essential Meta Tags for Social Media by CSS-Tricks. Basically, they're the stuff that you see when you embed them to social media.

The JavaScript Part

To get started, create a function. It should have the same name as the one in the onclick part of the <button> placed earlier. All the stuff will be placed inside the function.

To get the values inside the <textarea>, we'll be using document.getElementById('id-of-the-textarea').

Then, create a variable that essentially acts as a template and add the values of the previous variables to it. Use backticks(``) instead of quotes('' or "") on adding values so that it's easier to manage (you won't need to add \n on every newline & you can add variables by using ${variable-name}).

The location variable

You're probably confused about how the location variable works. Let me explain.

Let's say that for example, your site is inside a folder. Here, the location should have a value of "../". Let's also say that you have a main.css stylesheet but in the root folder. To access that folder in the template, all you have to do is put href="${location}main.css" in the JavaScript part. You won't ever need to proofread the generated file anymore!
